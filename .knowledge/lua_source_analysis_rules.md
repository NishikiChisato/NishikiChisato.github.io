# Lua 底层与源码分析规范 (Lua Source Code Analysis Guidelines)

## 1. 核心基调
- 这是一系列**硬核 (Hardcore)** 的源码解析文章。
- 读者对象是对 C 语言和编译器/解释器底层实现有一定了解的开发者。

## 2. 结构要求
每篇源码分析文章应当具备以下结构：
1. **背景介绍 (Intro)**：为什么要研究这个机制（例如 Table 的哈希冲突处理）？
2. **核心数据结构 (Data Structures)**：首先展示 C 语言的 `struct`，并详细解释每一个字段的内存意义。
3. **关键算法流程 (Algorithms/Execution Flow)**：解析核心函数的执行步骤（如 `luaH_get`, `luaH_newkey`）。
4. **内存占用与性能分析 (Memory & Performance)**：讨论空间复杂度、时间复杂度或缓存命中率。
5. **总结与心得 (Conclusion)**。

## 3. 语言与代码规范
- 对于 C 语言的变量名、宏定义、结构体名称，**必须使用反引号 (backticks)** 包裹，例如 `lua_State`, `TValue`, `alimit`。
- 在解释源码时，切忌贴大段未经处理的源码，应当**只截取核心代码段**，并在一旁加上注释，或者把大段代码拆分成几个小片段分别讲解。
- 善用无序列表梳理逻辑分支（如 `if-else` 的深层嵌套逻辑）。
